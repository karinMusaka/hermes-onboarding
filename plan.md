# Hermes オンボーディングパッケージ 実装計画

> **目的:** ターミナル未経験の中村さん(COO)が、1行コピペ → 対話型ウィザード → チュートリアルの流れで Hermes を使えるようになる

## 概要

```
ワンライナー (curl | bash)
  ↓
Phase 1: 自動インストール (Homebrew, Python, Hermes)
  ↓
Phase 2: 設定ウィザード (日本語、選択式、API キー設定)
  ↓
Phase 3: チュートリアル (実際に Hermes と会話しながら学ぶ)
  ↓
Phase 4: レベルアップミッション (日常業務で使う課題集)
```

## アーキテクチャ

- `install.sh` — ワンライナーで呼ばれるメインスクリプト
- `wizard.sh` — 日本語対話型の設定ウィザード
- `tutorial.sh` — ハンズオンチュートリアル（実際に hermes コマンドを叩く）
- `missions/` — レベルアップ用のミッションファイル群
- GitHub リポジトリ or Gist でホスト → `curl -sL URL | bash`

## 実装タスク

### Task 1: GitHub リポジトリ作成
- `starmusic-dev/hermes-onboarding` (private)
- 基本ディレクトリ構造

### Task 2: install.sh — メインインストーラー
- Homebrew 確認・インストール
- uv + Python 3.11
- Hermes クローン + setup-hermes.sh 実行
- 日本語メッセージ
- エラーハンドリング（Wi-Fi切れ、ディスク容量等）
- 完了後に wizard.sh を自動実行

### Task 3: wizard.sh — 設定ウィザード
- 3ステップの対話式設定
- API キー入力 + 接続テスト
- ユーザー情報（名前、メール、役職）
- Slack 連携（オプション）
- .env と config.yaml 生成
- user memory に初期プロフィール書き込み

### Task 4: tutorial.sh — チュートリアル
- 5レッスン構成
- 実際に hermes コマンドを実行させる
- 各レッスン後に結果確認
- スキップ可能

### Task 5: ミッションシステム
- ~/.hermes/missions/ にミッション定義
- hermes mission list / hermes mission [番号] で表示
- 完了マーク機能（手動）
- Hermes の memory にミッション進捗を保存

### Task 6: テスト + ドキュメント
- 新しい Mac で install.sh が通ることを確認
- トラブルシューティングガイド
- 木原さん向け: 中村さんサポート用メモ
